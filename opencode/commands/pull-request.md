## Create Pull Request Workflow

Create a Pull Request targeting the specified branch using GitHub CLI (`gh`).

**Target Branch:** `$ARGUMENTS`

If no target branch is provided, default to `main`.

---

### Step 1: Validate Environment

Check that we're in a git repository and `gh` is available:

```bash
git rev-parse --is-inside-work-tree
gh auth status
```

If `gh` is not authenticated, report the error and stop.

### Step 2: Check Current Branch

```bash
git branch --show-current
```

- Store the current branch name.
- If already on the target branch, warn the user and ask if they want to continue (creating a PR from main to main doesn't make sense).

### Step 3: Check for Uncommitted Changes

```bash
git status --porcelain
git diff --cached --quiet
```

- If there are uncommitted changes, **warn the user** but continue.
- Recommend they commit or stash changes before creating the PR.

### Step 4: Ensure Branch is Pushed to Remote

```bash
git rev-parse --abbrev-ref @{u} 2>/dev/null
```

- If the branch has no upstream, push it:
  ```bash
  git push -u origin $(git branch --show-current)
  ```
- If push fails, report the error and stop.

### Step 5: Fetch Commits Since Branching from Target

```bash
git log --oneline origin/$ARGUMENTS..HEAD
git log --format="%s" origin/$ARGUMENTS..HEAD
```

- Store the list of commits for the PR body.
- Store the commit subjects for generating a title.

### Step 6: Generate PR Title

Based on the commits since branching from the target:

1. If there's only **one commit**, use its subject line as the title.
2. If there are **multiple commits**:
   - Look for a common prefix or pattern (e.g., all `feat:` commits).
   - If a pattern exists, use it as the title (e.g., "feat: add new authentication flow").
   - Otherwise, use a descriptive summary like "Changes from feature branch" or derive from the most significant commit.
3. Keep the title under 72 characters.

### Step 7: Generate PR Body

Create a comprehensive body using this template:

```markdown
## Summary

<1-3 sentence overview of what this PR does>

## Changes

<List of key changes made>

## Commits

<Auto-generated list from step 5>

\`\`\`
<commit log output>
\`\`\`

## Related Issues

<If any issue numbers were referenced in commits (e.g., #123, Fixes #456), list them here. If none, omit this section.>

## Testing

<Brief note on how to test these changes>
```

### Step 8: Create the Pull Request

```bash
gh pr create \
  --base $ARGUMENTS \
  --head $(git branch --show-current) \
  --title "<generated title>" \
  --body "<generated body>"
```

### Step 9: Report Results

Provide a summary:
- PR URL (from the `gh` output)
- Source branch → Target branch
- Number of commits included
- Any warnings (uncommitted changes, etc.)
